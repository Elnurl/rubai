### 2.2 BEHAVIORAL DATA COLLECTION POINTS

#### A. Real-Time Signals

**Voice Analysis:**
```
Input: User speaks to Coach
- Speech rate (hesitation = uncertainty)
- Tone pitch variation (excitement vs. resignation)
- Pause length (reflection vs. confusion)
- Word choice sentiment (positive/negative language)
Output: Emotional state + confidence level
```

**Text Patterns:**
```
Input: Message to Coach
- Emoji usage (engagement indicator)
- Message length (detail-oriented vs. rushed)
- Capitalization patterns (urgency)
- Question marks (uncertainty)
Output: Cognitive state + engagement level
```

**Temporal Patterns:**
```
Input: When does user interact?
- Peak activity hours (morning? evening?)
- Session duration
- Days of week preference
- Time gaps between interactions
Output: Energy pattern + optimal scheduling
```

**Task Behavior:**
```
Input: How does user interact with tasks?
- Task completion time vs. deadline
- Frequency of edits/revisions
- Task switching patterns
- Goal abandonment rate
Output: Work style + realistic capacity
```

## 4. AI Roadmap Generation - Enhanced Algorithm

### Current System:
- Basic task breakdown
- No behavioral adaptation

### Optimized System:

```
STEP 1: GOAL ANALYSIS
├─ What is the goal?
├─ What's the emotional investment? (voice sentiment)
└─ Is it intrinsic or extrinsic?

STEP 2: USER BEHAVIOR ASSESSMENT
├─ Energy pattern: When are they strongest?
├─ Cognitive load: How much capacity left?
├─ Procrastination risk: High/Medium/Low?
└─ Motivation type match: Does goal align with their driver?

STEP 3: BEHAVIORAL-ADAPTED ROADMAP
├─ IF high procrastination risk → smaller first step
├─ IF low energy pattern detected → suggest "momentum building" tasks first
├─ IF high cognitive load → reduce parallel goals
├─ IF purpose-driven → emphasize "why" in each step
└─ IF flow-state person → longer, uninterrupted blocks

STEP 4: TIMING OPTIMIZATION
├─ Schedule tasks during peak productivity hours
├─ Add buffer time (25% extra if pattern shows)
├─ Insert "reflection moments" for detail-oriented users
└─ Space out deadlines (not back-to-back for energy-conscious users)

STEP 5: MOTIVATION CUSTOMIZATION
├─ IF Autonomy-driven: Emphasize choice/control
├─ IF Mastery-driven: Break into skill-progression steps
├─ IF Purpose-driven: Connect to larger vision
└─ IF External-motivated: Add milestone celebrations/badges

OUTPUT: Personalized Roadmap
```

## 5. Smart Task Suggestion System

### Current Issue:
Coach suggests next steps generically. Should be behavioral-intelligent.

### Optimized Approach:

```
DECISION TREE FOR TASK SUGGESTION:

Is user in FLOW STATE?
├─ YES → Suggest extension of current task
│        "You're in the zone - keep going 30 more min?"
└─ NO → Is user PROCRASTINATING?
        ├─ YES, HIGH ANXIETY → "Let's start super small"
        │                      (5-min task, lowest barrier)
        ├─ YES, LOW ENERGY → "Recharge task" 
        │                    (something they find easy/fun)
        └─ NO → Is ENERGY HIGH?
                ├─ YES → Suggest COMPLEX/challenging task
                └─ NO → Suggest ROUTINE/maintenance task
```

## 6. Real-Time Behavioral Coaching

### New Feature: "Nudge Intelligence"

Instead of generic reminders, context-aware nudges:

```
Scenario 1: User hasn't opened app in 3 days
Pattern shows: Often abandons goals after "initial excitement"
Nudge type: RECONNECTION NUDGE
Message: "Hey, I noticed you're usually strongest on 
         Tuesday mornings. Want to check in then?"

Scenario 2: User working on 6 simultaneous goals
Completion rate: Down 40%
Nudge type: CAPACITY WARNING
Message: "You're crushing it, but 6 goals at once 
         historically leads to overwhelm. 
         Which 3 matter most right now?"

Scenario 3: User just completed hard task
Emotional tone: Relief + pride
Nudge type: MOMENTUM BUILDING
Message: "That took guts. You're in flow mode. 
         Want to tackle the next step while 
         you're riding this?"

Scenario 4: User marked task complete suspiciously fast
Historical pattern: Incomplete work
Nudge type: QUALITY CHECK
Message: "Nice work! Just confirming - 
         does this feel truly complete?"
```

## 7. Privacy & Ethical Considerations

### ✅ What to Track:
- Completion times
- Goal types
- Message sentiment (not content)
- Voice tone (not transcription)
- Time stamps
- Device type

### ❌ What NOT to Track:
- Full message content beyond sentiment
- Location GPS data (unless explicitly opted in)
- Social media integration without consent
- Private conversations outside goal context
- Biometric data

**Implementation:** All behavioral analysis happens on-device first, with option to disable. Clear user settings for each tracking type.
