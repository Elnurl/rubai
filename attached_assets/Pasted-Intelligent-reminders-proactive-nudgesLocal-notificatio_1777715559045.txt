These are the spots where the docs ask for more than what's there:

Reminders are local-only. If the OS kills the schedule or the user is on web, nothing fires. The vision implies a system that proactively reaches out — that means server-side push (Expo push tokens + a cron / scheduled job).

No historical reporting view. The Account tab shows the current snapshot of the behavioral profile, not "your completion rate over the last 8 weeks" or "your motivation trend chart." The docs talk about AI-generated behavioral reports as a deliverable.

Attachments are theatrical. The paperclip lets the user feel like they uploaded something, but the model never sees the file. If "uploaded data" is part of the workspace promise, this needs real handling (object storage + vision or document parsing).

No real billing. Plans screen is a UI preview. Monthly subscription is in the vision; right now nobody can actually pay.

No domain-specific tools. A car-buying user gets the same chat-and-plan system as an IELTS user. The vision in doc 2 implies the AI should be able to do finance math, not just talk about it. Same for skill-specific scoring.

No explicit success-probability surface. The roadmap shows risks; there's no "based on the last 4 weeks, your odds of hitting your milestone on time are X%."
